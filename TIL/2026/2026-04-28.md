## Key concepts learned

- **캐시(Cache)**: 한 번 받은 파일을 저장해뒀다가 같은 요청이 오면 재사용하여 속도를 빠르게 함
- **localhost**: 자신의 컴퓨터를 가리키는 주소, Tomcat 테스트 시 http://localhost:8080/ 형태로 사용
- **W3C와 브라우저**: W3C가 인터페이스처럼 웹 표준 규칙을 정하면 브라우저 회사들이 실제 구현함
- **MDN(Mozilla Developer Network)**: 모르는 HTML 태그나 CSS 속성을 검색할 수 있는 공식 문서 사이트
- **CSS 3가지 적용 방식**: 인라인(비효율), 인터널(한 파일), 익스터널(여러 파일 공통 - 실무에서 가장 많이 사용)
- **기본 폰트 크기**: 16px = 1rem (h1은 32px, h4는 16px)
- **DOM 트리**: 나중에 CSS와 JS에서 접근하여 스타일 변경, 내용 수정, 태그 추가/삭제에 사용됨
- **HTML 주석**: 브라우저에 표시되지 않는 메모용 코드로 코드 설명이나 임시 비활성화에 사용
- **img 태그**: src(경로), alt(대체 텍스트), width/height(크기) 속성을 가지며 빈 태그
- **특수문자**: HTML에서 직접 입력할 수 없는 문자를 코드로 표현 (&nbsp;, &gt;, &quot; 등)
- **iframe 태그**: 현재 페이지 안에 다른 HTML 페이지를 삽입하며 유튜브, 지도 등 외부 콘텐츠에 사용
- **dialog 태그**: HTML5 기본 모달 팝업으로 showModal()로 열고 method="dialog"로 자동 닫힘 처리
- **캐스케이딩(Cascading)**: DOM 트리 상위 요소의 스타일이 하위로 전달되며 하위에서 재정의하면 덮어씀
- **명시도(Specificity)**: 선택자마다 우선순위가 있으며 순서는 인라인 > ID > class > 태그 (0100, 0010, 0001)
- **!important**: 모든 가중치를 무시하고 최우선 적용하는 강제 규칙
- **CSS 계층 구조 선택자**: 자식(>), 자손( ), 인접 형제(+), 일반 형제(~) 선택자 구분
- **CSS 속성 선택자**: input[type="text"], a[href^="http"] 등으로 속성값 기반 선택
- **의사 클래스(Pseudo-class)**: a:hover, li:first-child, li:nth-child(2n) 등 특정 상태일 때 스타일 적용
- **의사 요소(Pseudo-element)**: p::first-letter, p::before, p::after 등 요소 특정 부분에 스타일 적용 (::사용)
- **박스 모델**: margin(바깥 여백) > border(테두리) > padding(안쪽 여백) > content 구조
- **box-sizing: border-box**: width/height에 border와 padding을 포함하며 실무에서 전체 적용 많이 함
- **margin 상쇄**: 블럭 요소의 위아래 margin이 겹치면 큰 값 하나만 적용됨
- **border-radius**: 모서리를 둥글게 처리하며 50%로 원형 생성 가능
- **display 속성**: block(블럭), inline(인라인), inline-block(가로 배치 + 크기 지정)으로 요소 성질 변경
- **position 속성**: relative(상대 위치), fixed(고정 위치), absolute(절대 위치) 등으로 배치 방식 결정
- **absolute 사용 시**: 부모에 position: relative 필수, 없으면 최상위(body)까지 올라가서 기준 잡음
- **z-index**: 요소의 쌓임 순서 결정하며 position이 static
