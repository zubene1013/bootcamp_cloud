## Key concepts learned

- **Java Platform**: SE(Standard Edition), EE(Enterprise Edition), ME(Micro Edition) 세 가지 에디션으로 구분되며, JDK 설치로 기본 플랫폼 구성 가능
- **JDK vs JRE vs JVM**: JDK는 개발 도구 포함, JRE는 런타임 환경, JVM은 바이트코드 실행 엔진으로 계층적 관계
- **컴파일과 실행**: .java 소스 코드 → javac로 컴파일 → .class 바이트코드 생성 → java로 실행
- **압축 파일 형식**: JAR(클래스 파일), WAR(웹 애플리케이션), EAR(JAR+WAR) 세 가지 형식 제공
- **식별자 규칙**: 클래스명은 첫 글자 대문자, 예약어 불가, 대소문자 구분
- **Git 워크플로우**: Working Directory → Staging Area → Local Repository → Remote Repository의 4단계 프로세스
- **Git 브랜치**: 독립적인 개발 흐름을 위해 브랜칭하고 병합하는 협업 방식
- **Git 되돌리기**: reset(로컬만 변경), revert(커밋으로 기록), restore(파일 복구) 방식 구분

## Commands / Configuration

### Java Compilation and Execution

```
javac Test.java          # 소스 코드 컴파일 → Test.class 생성
java Test                # 컴파일된 바이트코드 실행
```

### Java Format Types

```
%s : string
%c : char
%d : integer
%f : float
%b : boolean
```

### Git Basic Workflow

```
git init                 # 로컬 저장소 초기화
git add [file]          # 파일 스테이징
git commit -m "message" # 커밋 생성
git push                # 원격 저장소에 푸시
```

### Git Log and Diff

```
git log --oneline              # 한 줄씩 커밋 히스토리 표시
git log -p                     # 변경사항 상세 표시
git log --relative-date        # 경과 시간 기반 표시
git diff                       # 워킹 디렉토리 vs 저장소 비교
git diff --staged              # 스테이징 에어리어 vs 저장소 비교
git diff HEAD                  # 모든 변경사항 비교
git diff [Hash1] [Hash2]       # 특정 커밋 간 비교
```

### Git Reset and Restore

```
git reset --soft [commit id]   # 커밋만 취소
git reset --mixed [commit id]  # 커밋과 스테이징 취소
git reset --hard [commit id]   # 모든 변경사항 취소
git restore [file]             # 수정 내용 삭제
git restore --staged [file]    # 스테이징 취소
git commit --amend             # 직후 커밋 수정
```

### Git Branch Management

```
git branch [name]              # 새 브랜치 생성
git branch -m [name]           # 브랜치명 변경
git branch -M [name]           # 브랜치명 강제 변경
git branch -D [name]           # 브랜치 삭제
git switch [name]              # 브랜치 변경
git switch -c [name]           # 브랜치 생성 및 변경
git merge [name]               # 브랜치 병합
git merge -D [name]            # 병합 후 브랜치 삭제
```

### Git Configuration

```
git status                      # 저장소 상태 확인
.gitignore                      # 추적 제외 파일/폴더 지정
git rm --cached [file]          # 이미 추적되는 파일 추적 해제
```
