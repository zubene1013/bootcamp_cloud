## Key concepts learned

- **Generics (제네릭스)**: Type safety를 보장하고 compile time에 실제 타입으로 대체되는 기능, 클래스/인터페이스/메서드에 사용 가능하지만 static과 객체 생성에는 사용 불가
- **Collection API**: 동적 크기, 모든 타입 가능, 많은 built-in 메서드 제공하는 API
- **@FunctionalInterface**: 정확히 하나의 추상 메서드를 가져야 하며 람다 표현식으로 표현되어야 하는 인터페이스
- **Stream API**: 컬렉션에서 통계, 필터링, 정렬, 중복제거 등의 처리를 제공하는 API
- **Variable Arguments (가변인자)**: 메서드에서 매개변수 개수를 동적으로 받을 수 있으며 배열로 관리되는 기능

## Commands / Configuration

### Collection API 계층구조

```
Collection<E>                    Map<K,V>
    |                                |
    |                            HashMap<K,V>
Set<E>      List<E>
    |           |
HashSet<E>  ArrayList<E>
```

### Variable Arguments 사용법

```java
// 일반 메서드
public void a(int a) { }
a(10);

// 가변인자 메서드 → 배열로 관리
public void a2(int ... a) { }
a2(1);
a2(1, 2, 3);
```
