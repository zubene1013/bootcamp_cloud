## Key concepts learned

- **ECMAScript**: 자바스크립트의 표준 구문으로 EC6(ES6)부터 표준으로 사용되며 브라우저가 실행하는 스크립트 언어
- **Script 위치**: body 태그 끝부분에 script를 배치하는 것이 권장되는데, HTML이 먼저 로드된 후 실행되기 때문
- **console.log()**: 브라우저 개발자도구(F12)에 로그를 출력하는 디버깅 도구
- **기본형(Primitive) 데이터**: number, string, boolean, undefined, null, NaN 등으로 값 자체를 저장
- **false로 처리되는 5가지 값**: 0, "", null, undefined, NaN을 제외한 나머지는 true로 처리됨
- **var vs let**: var는 함수 스코프를 가져 블록 밖에서도 접근 가능하지만, let은 블록 스코프를 가져 블록 내에만 제한됨
- **기본형 vs 참조형 변수**: 기본형은 값을 복사하지만, 참조형은 주소를 복사하여 같은 객체를 가리킴
- **const 상수**: 한 번 선언하면 값을 변경할 수 없지만, 배열이나 객체의 요소값은 변경 가능
- **산술 연산자와 형변환**: + 연산자는 문자열 연결을 하지만, 다른 연산자(-, *, /)는 문자열을 자동으로 수치로 형변환
- **동등 연산자(==) vs 일치 연산자(===)**: ==는 값만 비교하고 ===는 타입과 값을 모두 비교하며, null 비교 시 반드시 === 사용
- **논리 연산자**: && 연산자는 값1이 true이면 값2를 선택하고, || 연산자는 값1이 true이면 값1을 선택
- **증감 연산자의 순서**: ++n은 먼저 증가 후 대입, n++은 먼저 대입 후 증가
- **switch문**: 내부적으로 === 사용하므로 타입이 다르면 일치하지 않음
- **do~while 반복문**: 조건을 확인하기 전에 최소 1회 실행을 보장
- **BOM(Browser Object Model)**: 브라우저 창이 열리면 자동으로 생성되는 window, screen, location 등의 객체
- **DOM(Document Object Model)**: HTML 파일이 브라우저에 로드될 때 자동으로 생성되는 객체로 document를 통해 HTML 태그에 접근 및 조작
- **JSON 객체**: JavaScript Object Notation으로 key-value 쌍으로 표현되며, value는 모든 데이터 타입(배열, 중첩 JSON, 함수 등) 가능
- **JSON 접근**: 점 표기법(person.name)과 괄호 표기법(person["name"]) 모두 사용 가능

## Commands / Configuration

### JavaScript 기본 문법

```javascript
// Script 적용 방식
<!-- 1) head 태그 안 -->
<head>
    <script>
        alert("hello");
        console.log("콘솔에 출력", 10, 20);
    </script>
</head>

<!-- 2) body 태그 끝부분 (권장) -->
<body>
    ...
    <script>
        alert("hello");
    </script>
</body>

<!-- 3) 외부 파일 연결 -->
<script src="external.js"></script>
```

### 데이터 타입 확인

```javascript
console.log(typeof 100);         // "number"
console.log(typeof "hello");     // "string"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (버그)
console.log(typeof NaN);         // "number" (주의)
```
