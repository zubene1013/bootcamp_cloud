## Key concepts learned

- **JS 적용 위치**: head 태그 안 / body 태그 끝(권장) / 외부 파일(.js) 연결
- **body 끝 권장 이유**: HTML이 먼저 로드된 후 JS가 실행되어야 태그 접근 가능
- **console.log()**: 브라우저 개발자도구(F12) 콘솔에 출력하는 함수
- **기본형(Primitive)**: 수치(정수/실수 구분 없음), 문자(문자/문자열 구분 없음), 논리(true/false), undefined, null, NaN
- **참조형(Reference)**: 배열[], JSON{}, 함수function(){}, 클래스 — 함수도 데이터로 처리 가능 (변수에 저장 가능) ⭐
- **typeof 주의사항**: null → "object", NaN → "number" 반환 (버그성 특징)
- **false로 처리되는 값 5가지**: 0, "", null, undefined, NaN — 나머지는 모두 true ⭐
- **var**: 변수명 중복 가능, 함수 스코프 — 블럭 밖에서도 접근 가능 (비권장)
- **let**: 변수명 중복 불가, 블럭 스코프 — 블럭 밖 접근 불가 (ES6 권장) ⭐
- **const**: 상수 선언, 재할당 불가 — 단, 배열/객체의 요소값 변경은 가능
- **기본형 변수**: 값을 복사 → 한쪽 변경해도 다른 쪽 영향 없음
- **참조형 변수**: 주소를 복사 → 한쪽 변경하면 다른 쪽도 영향 받음 ⭐
- **== vs ===**: == 은 값만 비교(타입 무시), === 은 값+타입 모두 비교 (권장) ⭐
- **null == undefined**: true 반환 주의 — undefined 비교는 반드시 === 사용
- **조건부 렌더링 &&**: 값1이 true면 값2 선택, false면 값1 선택
- **조건부 렌더링 ||**: 값1이 true면 값1 선택, false면 값2 선택 — 나중에 React에서 자주 활용
- **switch 내부**: === (identical) 사용 — 타입이 다르면 일치 안 함 주의
- **Window 객체**: 브라우저 창 관련 전역객체 — screen, history, location, navigator는 window 속성 (생략 가능)
- **BOM**: 브라우저를 열면 자동 생성되는 객체 모음 (Window, Screen, History, Location, Navigator)
- **DOM**: HTML 파일이 브라우저에 보여질 때 자동 생성되는 document 객체
- **JSON 객체**: {key:value} 형식, value에 수치/문자/배열/JSON/함수 모두 가능

## Commands / Configuration

### JS 적용 방법

```html
<!-- 1) head 태그 안 -->
<head>
    <script>
        alert("hello");
        console.log("콘솔 출력", 10, 20);
    </script>
</head>

<!-- 2) body 태그 끝 (권장) -->
<body>
    ...
    <script>
        alert("hello");
    </script>
</body>

<!-- 3) 외부 파일 연결 -->
<script src="external.js"></script>
```

### 데이터 종류 및 typeof

```javascript
console.log(typeof 100);         // "number"
console.log(typeof "hello");     // "string"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object"  ← 주의!
console.log(typeof NaN);         // "number"  ← 주의!
console.log(typeof []);          // "object"
console.log(typeof {});          // "object"
console.log(typeof function(){}); // "function"
```

### var vs let vs const

```javascript
// var - 함수 스코프 (비권장)
if(true) { var n = 100; }
console.log(n);  // 100 (블럭 밖에서도 접근됨!)

// let - 블럭 스코프 (권장)
if(true) { let n2 = 100; }
// console.log(n2);  // 오류!

// const - 상수
const NUM = 10;
// NUM = 20;         // 오류!
const NUMS = [10, 20, 30];
NUMS[0] = 100;       // 가능 (요소값 변경)
// NUMS = [1,2,3];   // 불가 (주소 변경)
```

### 기본형 vs 참조형 변수

```javascript
// 기본형 - 값 복사
let n = 10;
let n2 = n;
n2 = 20;
console.log(n, n2);  // 10, 20 (서로 영향 없음)

// 참조형 - 주소 복사
let x = [10, 20, 30];
let x2 = x;
x2[0] = 100;
console.log(x, x2);  // 둘 다 [100, 20, 30]
```

### 비교 연산자

```javascript
console.log("10" == 10);   // true  (값만 비교)
console.log("10" === 10);  // false (값+타입 비교)

let x = null;
console.log(x == undefined);   // true  (주의!)
console.log(x === undefined);  // false
```

### 논리 연산자 (조건부 렌더링)

```javascript
// && : 값1이 true면 값2, false면 값1
console.log(10 && "홍길동");   // "홍길동"
console.log(NaN && "홍길동");  // NaN

// || : 값1이 true면 값1, false면 값2
console.log(10 || "홍길동");   // 10
console.log(NaN || "홍길동");  // "홍길동"
```

### 조건문 / 반복문

```javascript
// if ~ else if ~ else
let num = Number.parseInt(prompt("점수 입력"));
if(num >= 90) { console.log("A학점"); }
else if(num >= 80) { console.log("B학점"); }
else { console.log("F학점"); }

// switch (=== 사용)
switch(x) {
    case 100: console.log("100"); break;
    default:  console.log("default"); break;
}

// for / while / do-while
for(let i = 0; i < 5; i++) { console.log(i); }

let i = 0;
while(i < 5) { console.log(i); i++; }

do { console.log(i); i++; } while(i < 5);
```

### JSON 객체

```javascript
let person = {
    name: "홍길동",
    age: 20,
    phone: ["010", "011"],
    hobby: { sport: "축구", music: "기타" },
    getName: function() { return this.name; }
};

console.log(person.name);       // "홍길동"
console.log(person["name"]);    // "홍길동"
console.log(person.getName());  // "홍길동"
```
