## Key concepts learned

- **DB vs DBMS**: DB는 데이터 자체이고, DBMS는 DB를 관리하는 소프트웨어(예: MySQL, Oracle, MS-SQL)
- **SQL 분류**: CRUD 작업으로 분류 - Create(insert), Read(select), Update(update), Delete(delete)
- **SELECT 문 실행 순서**: FROM → WHERE → SELECT 순서로 실행되며, 이 순서를 이해하는 것이 중요
- **NULL 값**: 0이나 공백과 다른 개념으로, IS NULL 또는 IS NOT NULL로만 비교 가능
- **제약조건(Constraints)**: 데이터 무결성을 위해 컬럼에 설정하는 규칙
- **BETWEEN**: 경계값을 포함하는 범위 검색
- **IN**: 목록 중 하나의 값과 일치하는지 검색
- **LIKE**: 패턴 매칭으로 '%'와 '_' 와일드카드 사용 가능
- **ORDER BY**: 결과를 정렬하며, ASC(오름차순)가 기본값이고 DESC로 내림차순 설정 가능

## Commands / Configuration

### SELECT 문 기본 구조

```sql
SELECT 컬럼      -- 3
FROM 테이블      -- 1
WHERE 조건;      -- 2
```

### SELECT 다양한 사용법

```sql
SELECT *                    -- 전체 컬럼
SELECT empno, ename         -- 특정 컬럼
SELECT sal * 1.1            -- 산술연산
SELECT DISTINCT deptno      -- 중복 제거
SELECT ifnull(comm, 0)      -- NULL이면 0으로 대체
SELECT sal AS "월급"         -- 별칭(alias)
SELECT sal 월급              -- AS 생략 가능
```

### NULL 값 처리

```sql
-- ✅ 올바른 NULL 비교
WHERE comm IS NULL
WHERE comm IS NOT NULL

-- ❌ 작동 안 함
WHERE comm = NULL

-- NULL 포함 연산 결과는 항상 NULL
SELECT ifnull(comm, 0) FROM emp;
```

### WHERE 조건절 - 비교 연산자

```sql
WHERE sal = 1000
WHERE sal >= 1000
WHERE ename = 'SMITH'          -- MySQL은 대소문자 무시
WHERE hiredate > '1980-12-17'  -- 날짜도 '' 감싸기
```

### WHERE 조건절 - BETWEEN

```sql
WHERE sal BETWEEN 1000 AND 2000
```

### WHERE 조건절 - IN

```sql
WHERE deptno IN (10, 20, 30)
WHERE ename IN ('SMITH', 'FORD')
```

### WHERE 조건절 - LIKE 패턴 검색

```sql
WHERE ename LIKE '%A%'    -- A 포함
WHERE ename LIKE 'A%'     -- A로 시작
WHERE ename LIKE '%A'     -- A로 끝남
WHERE ename LIKE '_A'     -- 두 글자이고 A로 끝남
WHERE ename LIKE '___A%'  -- 네 번째 글자가 A
```

### WHERE 조건절 - 논리 연산자

```sql
WHERE sal > 1000 AND deptno = 20
WHERE sal > 1000 OR deptno = 20
WHERE NOT sal BETWEEN 1000 AND 2000
WHERE NOT deptno IN (10, 20)
```

### ORDER BY 정렬

```sql
ORDER BY sal           -- 오름차순 (기본값)
ORDER BY sal ASC       -- 오름차순 명시
ORDER BY sal DESC      -- 내림차순
ORDER BY sal ASC, deptno DESC  -- 다중 정렬
```

### 전체 SELECT 작성 순서

```sql
SELECT ...
FROM ...
WHERE ...       -- 선택
ORDER BY ...;   -- 선택, 항상 마지막
```
