## Key concepts learned

- **Git Workflow**: Working Directory → Staging Area → Local Repository → Remote Repository로 이어지는 버전 관리 프로세스
- **Commit 수정**: `--amend` 옵션으로 방금 커밋한 내용을 즉시 수정 가능
- **파일 상태 추적**: `git diff`로 Working Directory, Staging Area, Repository 간의 변경사항 비교
- **Reset vs Revert**: `reset`은 커밋 이력을 삭제하고 `revert`는 새로운 커밋으로 되돌림
- **브랜치 관리**: 새로운 작업 흐름을 위해 브랜치를 생성하고 병합할 수 있음
- **Git 내부 구조**: `.git/refs/heads/`에 각 브랜치의 최신 커밋 정보가 저장되고, `.git/objects/`에서 해시값으로 파일 관리
- **.gitignore**: Git이 추적하지 않을 파일/폴더를 지정하여 `git add .` 실행 시 제외

## Commands / Configuration

### Git Basic Commands

```javascript
git init                          // Create local repository
git add                           // Stage files for commit
git commit -m "message"           // Create version
git commit -am "message"          // Add and commit tracked files in one step
git commit --amend                // Modify immediately after committing
git log                           // Check version history
git log --oneline                 // One commit per line
git log -p                        // Print changes with diff
git log --relative-date           // Print elapsed time since commit
git status                        // Check repository status
git push                          // Push local branch to remote repository
```

### Git Diff Commands

```javascript
git diff                          // Compare working directory and repository
git diff --staged                 // Compare staging area and repository
git diff HEAD                     // Compare working directory and staging area with repository
git diff Hash1 Hash2              // Compare between commits
```

### Git Restore Commands

```javascript
git restore                       // Delete modified content
git restore --staged              // Cancel add (unstage files)
```

### Git Reset Commands

```javascript
git reset --soft <commit id>      // Cancel commit only
git reset --mixed <commit id>     // Cancel add and commit
git reset --hard <commit id>      // Cancel add, commit and delete modified content
```

### Git Branch Commands

```javascript
git branch (name)                 // Create new branch
git branch -m (name)              // Rename branch (normal)
git branch -M (name)              // Force rename branch (overwrite)
git branch -D (name)              // Delete branch
git switch (name)                 // Change branch
git switch -c (name)              // Create and change branch
git merge (name)                  // Merge branches
git merge -D (name)               // Delete branch after merge
```

### Git Ignore

```javascript
git rm --cached <filename>        // Remove already tracked file from cache
```
