## Key concepts learned

- **웹 아키텍처**: 클라이언트와 서버가 요청/응답을 주고받는 구조로, 서버는 클라이언트 요청을 처리하고 응답하는 역할을 함
- **정적 컴포넌트**: 웹서버에서 HTML 파일을 찾아 응답하는 방식으로 항상 동일한 화면을 보여줌
- **동적 컴포넌트**: 서버에서 요청할 때마다 실행하여 실행 결과를 HTML로 만들어 응답하므로 매번 결과가 달라짐
- **브라우저 캐시**: 한 번 다운로드한 파일은 캐시에 저장되어 같은 요청 시 캐시된 파일을 사용함
- **웹 표준**: 모든 브라우저에서 동일하게 동작하도록 W3C에서 정한 규칙 (1993년 창설)
- **포트번호**: HTTP(80), HTTPS(443), Tomcat(8080), MySQL(3306), Oracle(1521), SSH(22)
- **HyperText**: 링크로 연결되어 선택 시 새 화면으로 이동하는 텍스트
- **Markup Language**: 태그로 구성되어 특정 의미를 가지는 언어
- **DOM 트리**: 브라우저가 HTML을 읽어 계층적인 트리 구조로 만든 것으로, 태그, 속성, 텍스트가 모두 노드가 됨
- **DOCTYPE 선언**: 브라우저에게 HTML5 문법으로 처리하라고 알려주는 선언으로 파일 맨 위에 작성해야 함
- **전역 속성**: 모든 태그에 사용 가능한 속성(id, class, name, style)
- **지역 속성**: 특정 태그에서만 사용 가능한 속성(href, value 등)
- **커스텀 속성**: data-속성명="값" 형식으로 임의로 지정 가능한 속성
- **빈 태그**: 종료 태그가 없는 태그로 `<br>`, `<img>`, `<input>`, `<hr>` 등이 있음
- **CSS 설정 3가지**: 인라인(style 속성), 인터널(`<style>` 태그), 익스터널(`<link>` 태그)
- **JS 설정 2가지**: 내부(`<script>` 태그), 외부(`<script src="">` 태그)
- **인라인 태그**: 가로 방향으로 배치되고 필요한 만큼의 공간만 차지하는 태그 (width/height 적용 안 됨)
- **블럭 레벨 태그**: 세로 방향으로 배치되고 전체 가로 공간을 차지하는 태그 (width/height 적용됨)
- **기본 폰트 크기**: 16px (= 1rem) ⭐
- **a 태그의 target="_blank"**: 새 탭에서 링크를 열도록 지정
- **a 태그 기본 색상**: 방문 전(파랑), 방문 후(보라), 활성화(빨강) — CSS로 변경 가능
- **내부 링크**: 같은 페이지 내 특정 위치로 이동하기 위해 id 속성과 href="#id명" 사용
- **표 셀 병합**: colspan으로 열 병합(가로), rowspan으로 행 병합(세로)
- **순서 있는 목록 start 속성**: 원하는 번호부터 시작하도록 지정 가능
- **시맨틱 태그**: `<div>` 대신 태그 이름 자체에 의미를 포함하는 HTML5 구조 태그 사용
- **form 태그**: 사용자 입력 데이터를 서버로 전송하는 태그, action(서버 주소)과 method(get/post) 속성 사용
- **GET vs POST**: GET은 URL에 데이터 노출(조회용), POST는 body에 숨겨서 전송(로그인, 회원가입 등)
- **W3C 표준은 스펙(spec)**: 자바의 인터페이스 개념과 동일하며, 실제 구현은 브라우저 벤더가 함

## Commands / Configuration

### HTML 기본 구조

```html
<!DOCTYPE html>
<html>
  <head>
    <title>페이지 제목</title>
  </head>
  <body>
    <!-- 화면에 표시될 내용 -->
  </body>
</html>
```

### 요청 URL 구조

```
http://서버IP:포트번호/자원
예) http://localhost:8080/index.html
```

| 포트 | 설명 |
|------|------|
| 80   | HTTP (기본 웹) — 데이터 노출됨 |
| 443  | HTTPS (보안 웹) — 데이터 암호화됨 |
| 8080 | Tomcat |
| 3306 | MySQL |
| 1521 | Oracle |
| 22   | SSH (리눅스 원격 접속) |

### 웹 표준 기술 역할

| 기술       | 표준                    | 역할 |
|------------|-------------------------|------|
| 구조       | HTML5                   | 데이터 구조 (뼈대) |
| 스타일     | CSS3                    | 스타일 (디자인) |
| 동작       | ECMAScript5 (JavaScript) | 동적 처리 (기능) |

- 브라우저 호환성 확인: https://developer.mozilla.org

### CSS 설정 방법 3가지

```html
<!-- 1) 인라인 -->
<p style="color: red;">Hello</p>

<!-- 2) 인터널 -->
<style>
    p { color: red; }
</style>

<!-- 3) 익스터널 -->
<link rel="stylesheet" href="style.css">
```

| 방식 | 위치 | 특징 |
|------|------|------|
| 인라인(Inline) | 태그 안 style 속성 | 해당 태그에만 적용 |
| 인터널(Internal) | `<head>` 안 `<style>` 태그 | 해당 HTML 파일 전체 적용 |
| 익스터널(External) | 별도 .css 파일 연결 | 여러 파일에 공통 적용 가능 |

### JS 설정 방법 2가지

```html
<!-- 내부(internal) -->
<script>
    alert("경고창");
</script>

<!-- 외부(external) -->
<script src="external.js"></script>
```

### 인라인 vs 블럭 레벨

모든 HTML 태그는 box로 되어 있음 ⭐

| 구분 | 인라인 레벨 | 블럭 레벨 |
|------|------------|-----------|
| 배치 | 가로 배치 | 세로 배치 |
| 너비 | 컨텐츠 크기만큼만 | 브라우저 전체 너비 차지 |
| width/height | 적용 안 됨 | 적용 됨 |
| 대표 태그 | `<span>`, `<a>` | `<p>`, `<div>`, `<table>`, `<ul>`, `<ol>` |

### 제목 태그 — h1~h6

```html
<h1>가장 큰 제목 (32px)</h1>
<h4>중간 제목 (16px = 기본폰트)</h4>
<h6>가장 작은 제목</h6>
```

### 단락 / 구분선 / 줄바꿈 — p, hr, br

```html
<p>단락 내용</p>   <!-- 단락(paragraph) -->
<hr>               <!-- 수평선, 종료 태그 없음 -->
<br>               <!-- 줄바꿈(\n과 동일), 종료 태그 없음 -->
```

### 인라인 텍스트 그룹핑 — span

```html
<!-- 특정 부분만 스타일 적용할 때 사용 -->
<p>안녕하세요. <span style="color:red;">홍길동</span>님</p>
```

### 링크 — a (anchor)

```html
<!-- 외부 링크 (현재 창) -->
<a href="http://www.google.com">구글</a>

<!-- 외부 링크 (새 탭) -->
<a href="http://www.google.com" target="_blank">구글 새탭</a>

<!-- 내부 링크 (같은 페이지 내 특정 위치로 이동) -->
<a href="#html">HTML로 이동</a>
...
<a id="html">여기서부터 HTML</a>
```

- `href`: 이동할 주소
- `target="_blank"`: 새 탭에서 열기
- 기본 색상: 방문 전(파랑), 방문 후(보라), 활성화(빨강) — CSS로 변경 가능

### 표 — table

```html
<table border="1">
    <thead>
        <tr>
            <th>번호</th>
            <th>이름</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>홍길동</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td>총합</td>
            <td>10000</td>
        </tr>
    </tfoot>
</table>
```

| 태그 | 역할 |
|------|------|
| `table` | 표 전체 감싸기 |
| `tr` | 행(row) |
| `th` | 헤더 셀 (굵게, 가운데 정렬) |
| `td` | 일반 데이터 셀 |
| `thead` / `tbody` / `tfoot` | 표 영역 그룹핑 (JS/CSS 적용 목적) |

```html
<!-- 열 병합(가로): colspan -->
<th colspan="2">전화번호</th>

<!-- 행 병합(세로): rowspan -->
<th rowspan="2">전화번호</th>
```

### 목록 — ol, ul, li

```html
<!-- 순서 없는 목록 -->
<ul>
    <li>항목1</li>
    <li>항목2</li>
</ul>

<!-- 순서 있는 목록 -->
<ol>
    <li>첫 번째</li>
    <li>두 번째</li>
</ol>

<!-- 5부터 시작 -->
<ol start="5">
    <li>다섯 번째</li>
</ol>
```

- 블럭 레벨
- 주로 메뉴 만들 때 사용 (CSS로 인라인으로 변경해서 가로 배치)

### 블럭 / 인라인 그룹핑 — div, span

| 태그 | 특징 |
|------|------|
| `div` | 태그들을 그룹핑 — 블럭 레벨, width/height 적용됨 |
| `span` | 텍스트(값)를 그룹핑 — 인라인 레벨, width/height 무시됨 |

```html
<div>블럭 영역 (width/height 적용됨)</div>
<span>인라인 영역 (width/height 무시됨)</span>
```

### 시맨틱 태그 — header, nav, section, article, aside, footer

`<div>` 대신 용도에 맞는 의미 있는 태그를 사용

| 태그 | 역할 |
|------|------|
| `header` | 사이트 소개, 로고, 메인 메뉴 |
| `nav` | 내비게이션 메뉴, 링크 모음 |
| `section` | 페이지의 논리적 영역(본문 내용) |
| `article` | 독립적인 콘텐츠 단위 |
| `aside` | 부가 정보, 사이드바 |
| `footer` | 저작권, 작성자 정보, 하단 링크 |

```html
<header>
    <h1>사이트 제목</h1>
    <nav>
        <ul>
            <li><a href="1.html">메뉴1</a></li>
        </ul>
    </nav>
</header>
<section>
    <article>
        <p>본문 내용</p>
    </article>
</section>
<aside>
    <p>부가 정보</p>
</aside>
<footer>
    <p>저작권 정보</p>
</footer>
```

### 사용자 입력 태그 — form

```html
<form action="target.html" method="get">
    <input type="text" name="username">
    <input type="submit" value="전송">
</form>
```

- `action`: 데이터를 전송할 서버 주소
- `method`: 전송 방식 (`get` / `post`)
- 데이터 전달 포맷: `key=value&key=value` (key = name 속성값, value = 입력값)

#### GET vs POST

| 항목 | GET | POST |
|------|-----|------|
| 데이터 위치 | URL에 포함됨 (노출됨) | 요청 body에 포함 (숨겨짐) |
| 보안 | 낮음 | 높음 |
| URL 예시 | `target.html?age=10&name=홍길동` | `target.html` |
| 용도 | 조회 | 로그인, 회원가입 등 |

### input 타입 종류

```html
<!-- 텍스트 -->
<input type="text" name="username" placeholder="이름입력">

<!-- 비밀번호 -->
<input type="password" name="passwd">

<!-- 라디오 버튼 (단일 선택) -->
<input type="radio" name="gender" value="남">남
<input type="radio" name="gender" value="여" checked>여

<!-- 체크박스 (다중 선택) -->
<input type="checkbox" name="fruit" value="사과" checked>사과
<input type="checkbox" name="fruit" value="바나나">바나나

<!-- 숫자 -->
<input type="number" name="count" min="0" max="5" step="1">

<!-- 날짜 -->
<input type="date" name="m_date">

<!-- 이메일 (@ 필수 검증) -->
<input type="email" name="m_email">

<!-- 파일 업로드 -->
<input type="file" name="m_file" accept="image/*" multiple>

<!-- 숨김 필드 (사용자에게 안 보임, 서버에 값 전달) -->
<input type="hidden" name="address" value="seoul">

<!-- 슬라이더 -->
<input type="range" name="m_range" min="1" max="10">

<!-- 전송 버튼 -->
<input type="submit" value="전송">

<!-- 초기화 버튼 -->
<input type="reset">

<!-- 일반 버튼 (submit 안 됨) -->
<input type="button" value="버튼">
```

#### input 주요 속성

| 속성 | 설명 |
|------|------|
| `value` | 기본값 설정 |
| `readonly` | 읽기 전용 (수정 불가) |
| `required` | 필수 입력 (비어있으면 전송 안 됨) |
| `autofocus` | 페이지 로드 시 자동 포커스 |
| `placeholder` | 입력 힌트 텍스트 |
| `checked` | 체크박스/라디오 기본 선택 |
| `name` | 서버 전달 시 key값 |

### 기타 입력 태그 — select, textarea, label

```html
<!-- 드롭다운 선택 -->
<select name="cars">
    <option value="volvo">Volvo</option>
    <option value="audi">Audi</option>
</select>

<!-- 여러 줄 텍스트 (기본값은 태그 몸체에 작성) -->
<textarea name="message" rows="10" cols="30">기본 내용</textarea>

<!-- label: input과 연결하여 클릭 영역 확장 -->
<label for="name">이름:</label>
<input type="text" id="name" name="username">
```
