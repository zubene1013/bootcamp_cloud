## Key concepts learned

- **중첩 클래스(Nested Class)**: 클래스 안에 정의된 또 다른 클래스로, Member Inner, Local Inner, Static Inner 세 가지 종류가 있음
- **익명 클래스(Anonymous Class)**: 이름이 없는 일회용 클래스로 파일이 필요 없고 한 번만 사용됨
- **람다 표현식(Lambda Expression)**: 익명 클래스를 더 간결하게 작성한 방식으로 메서드에만 집중함
- **@FunctionalInterface**: 추상메서드가 정확히 1개만 있어야 하며, default 메서드는 허용되는 애노테이션
- **String 리터럴 vs new**: 같은 문자열 리터럴은 한 번만 생성되지만 new로 생성하면 매번 새로운 객체가 생성됨
- **StringBuilder**: 문자열을 효율적으로 연결하기 위한 유틸리티 클래스로 최종적으로 toString()으로 String으로 변환함
- **Calendar 싱글톤 패턴**: getInstance() 메서드로 인스턴스를 얻으며, 처음 호출 시에만 객체를 생성하고 이후 같은 객체를 반환함

## Commands / Configuration

### 중첩 클래스 종류별 사용법

```java
// Member Inner 클래스
public class Outer {
    class Inner { }
}
Outer o = new Outer();
Outer.Inner i = o.new Inner();

// Local Inner 클래스
public class Outer {
    public void a() {
        class Inner { }
    }
}

// Static Inner 클래스
public class Outer {
    static class Inner {
        static int num = 10;
    }
}
Outer.Inner i = new Outer.Inner();
```

### 익명 클래스 vs 람다 표현식

```java
// 익명 클래스
Flyer f = new Flyer() {
    @Override
    public void fly() { System.out.println("날다"); }
};

// 람다 표현식
Flyer f = () -> System.out.println("날다");
```

### 람다 표현식 4가지 문법

```java
// 1. 파라미터 없음, 리턴 없음
Flyer f = () -> System.out.println("fly()");

// 2. 파라미터 있음, 리턴 없음
Flyer2 f = (int n) -> System.out.println("fly()" + n);
Flyer2 f2 = (n) -> System.out.println("fly()" + n);
Flyer2 f3 = n -> System.out.println("fly()" + n);
Flyer2 f4 = n -> System.out.println("fly()" + n);

// 3. 파라미터 없음, 리턴 있음
Flyer4 f = () -> { return "Hello"; };
Flyer4 f = () -> "Hello";

// 4. 파라미터 있음, 리턴 있음
Flyer5 f = (int n, int n2) -> { return n + n2; };
Flyer5 f = (n, n2) -> { return n + n2; };
Flyer5 f = (n, n2) -> n + n2;
```

### @FunctionalInterface 규칙

```java
@FunctionalInterface
public interface Flyer {
    public abstract void fly();  // 추상메서드 반드시 1개
    default void land() { }      // default 메서드는 허용
}
```

### String 생성 방식 비교

```java
// 리터럴 - 같은 문자열은 한 번만 생성
String s = "hello";
String s2 = "hello";
(s == s2) // true

// new - 매번 새로운 객체 생성
String s = new String("hello");
String s2 = new String("hello");
(s == s2) // false
```

### StringBuilder 사용법

```java
StringBuilder sb = new StringBuilder();
sb.append("hello");
sb.append(" world");
String s = sb.toString();
```

### Calendar 싱글톤 패턴

```java
Calendar cal = Calendar.getInstance();

// 싱글톤 패턴 구현 원리
public class Calendar {
    private static Calendar cal;
    public static Calendar getInstance() {
        if(cal == null) cal = new Calendar();
        return cal;
    }
}
```
