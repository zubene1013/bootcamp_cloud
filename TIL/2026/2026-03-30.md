## Key concepts learned

- **Variable**: Declaration으로 변수를 선언하고, Initialization으로 값을 할당한 후, 필요에 따라 값을 변경하며 사용
- **Primitive vs Reference**: Primitive 타입은 실제 값을 저장하고 Reference 타입은 메모리 주소를 저장
- **Local variable**: Stack 메모리에 저장되며 반드시 초기화 후 사용해야 하고, 메서드 실행 시 생성되어 종료 시 삭제됨
- **Instance variable**: Heap 메모리에 저장되며 자동으로 기본값으로 초기화되고, new마다 별도로 생성됨
- **Stack vs Heap**: Stack은 Local variable을 저장하며 자동 관리되고 빠르며, Heap은 실제 객체를 저장하고 GC로 관리되며 느림
- **new Object Create**: Heap에 객체를 생성하고 주소를 반환하며, new마다 별도의 객체가 생성되고 참조되지 않으면 GC 대상이 됨
- **Constant**: final 키워드를 사용하며 값을 변경할 수 없고 UPPERCASE로 작성
- **Automatic Type Conversion**: byte → short → int → long → float → double 순서로 자동 변환되며, int보다 작은 타입의 연산 결과는 int로 반환되고 작은 타입 + 큰 타입은 큰 타입으로 반환
- **Operator**: 산술, 대입, 비교, 논리 연산자가 있으며 비교와 논리 연산 결과는 항상 boolean
- **Increment / Decrement Operator**: 단독 사용 시 전/후 결과가 같지만 다른 연산자와 함께 사용하면 결과가 다름
- **Ternary Operator**: (condition) ? true : false 형식으로 조건에 따라 값을 선택
- **Scanner**: System.in에서 입력을 받기 위해 Scanner 객체를 생성하고, nextInt() 등으로 입력받은 후 close()로 리소스 해제

## Commands / Configuration

### Scanner 기본 사용법

```java
import java.util.Scanner;

Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
sc.close();
```

### Variable 선언 및 사용

```java
int num;                          // Declaration
num = 10;                         // Initialized
System.out.println(num + 10);     // Use
num = 20;                         // Change
```

### Constant 선언

```java
final int MAX_VALUE = 100;
```

### Ternary Operator

```java
variable = (condition) ? true : false;
```

### Automatic Type Conversion

```java
byte b = 10;
int i = b;                        // byte -> int
int result = 5 + 10.5;            // int + double -> double
```
