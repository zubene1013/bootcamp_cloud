## Key concepts learned

- **? 바인딩 변수 (PreparedStatement)** - SQL에서 값을 나중에 넣기 위한 자리 표시자로, setInt()/setString() 등으로 채우며 SQL 인젝션 방지 및 코드 안전성 향상
- **커스텀 예외 (Custom Exception)** - 특정 상황에 맞는 예외를 직접 만들어 Exception을 상속받고, throw/throws를 통해 예외 흐름 관리
- **throw vs throws** - throw는 예외를 직접 발생시키고 throws는 예외를 위로 전달하겠다고 선언
- **RecordNotFoundException** - 삭제하려는 데이터가 없을 때 발생하는 커스텀 예외 (executeUpdate() 결과가 0일 때)
- **LIKE 검색** - "%" 와일드카드를 이용해 한 글자만 같아도 모든 행을 검색
- **BETWEEN + YEAR() 함수** - 입학년도 범위 검색에서 YEAR() 함수로 날짜에서 년도만 추출
- **StringBuilder + IN절** - 쉼표 구분자로 여러 값을 동적으로 조립해 다중 검색 구현
- **UPDATE + IN절** - 여러 행을 한번에 수정하고 executeUpdate()로 변경된 행 수 반환
- **CASE WHEN** - SQL의 if-else 구조로 조건별 UPDATE를 입력값 없이 자동 처리
- **다중 JOIN** - 여러 테이블을 연결해 조회하며 테이블 별칭 사용으로 컬럼 구분
- **scan.nextInt() vs scan.nextLine()** - nextInt() 다음 nextLine() 사용 시 개행문자(\n) 제거를 위해 nextLine() 먼저 호출
- **pad() 메서드** - 한글(2칸)/영문(1칸) 다르게 계산해 같은 너비로 정렬 (String.format 대체)
- **메서드 분리** - 반복되는 코드를 메서드로 뽑아 코드 중복 제거, 가독성 향상, 유지보수 편리
- **GradeDTO** - DB에서 받은 점수를 toString()에서 학점으로 계산해 표시
